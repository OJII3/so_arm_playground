# so_arm_playground

## プロジェクトの運用方針

このリポジトリは SO-101 のプロジェクトをまとめるモノレポ.

- 大きな環境依存は `flake.nix` で管理する. `nix develop` や `direnv` でプロジェクト限定のコマンドを使用できる.
- 個々のプロジェクトはディレクトリを設ける. プロジェクトごとに `README.md` を置いて、プロジェクトの技術構成を記載する.
- リポジトリルートに、できるだけ個々のプロジェクトのコードや設定・依存を置かないようにする.
- 複数のフレームワークを使用するプロジェクトは、フレームワークごとにディレクトリを分ける. 関心の分離・モジュール化を意識し、疎結合にする.
- Python のプロジェクトは `uv` で管理する.
- Unity のプロジェクトは `uloop` CLI で操作する. uloop の skill を使用する.
- ROS 2 を使用する際は `nix-ros-overlay` を利用し、ワークスペースごとリポジトリ管理下に置く.

## README に書くこと

- プロジェクトの一覧と各プロジェクトの概要
- 各プロジェクトの代表的な起動コマンド

## Git の運用

- コミットはと～っても小さく切る. 未コミットの変更を放置しない.
- タスク完了後は PR を作成する. PR を出す前に、変更に関連する箇所をできる限り静的・動的に検証する.
